# Flow used for project

1. Documenting initial idea
### At a high, user use level
2. Talking through briefly
3. Breaking into parts
4. Do user diaries of specific features
5. Go to 2
### At a lower, technical design level
6. Choose a feature
7. Ask about frameworks which could be used to implement the thoughts discussed before
8. Develop a plan for implementing that portion
9. Go to 6
